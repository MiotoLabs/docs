---
title: Builder und Blöcke
---

Öffnest du eine Anwendung, öffnet sich ihr Builder.

### **Builder-Interface**

1. Das Builder-Interface besteht im wesentlichen aus dem "Canvas", auf dem du deine Mioto-Anwendungen erstellst.
2. Auf dem Canvas fügst du Blöcke hinzu und verbindest diese miteinander, um deiner Anwendung die gewünschte Funktionalität zu geben.

### **Blöcke**

1. Blöcke sind die elementaren Bausteine deiner Anwendung. Jeder Block repräsentiert eine spezifische Aktion, die deine Anwendung ausführen kann.
2. Die grundlegenden Aktionen von Blöcken umfassen das Stellen von Fragen, das Sammeln und Austauschen von Daten sowie das Generieren von Dokumenten.

### **Start- und Endblock**

1. Jede Anwendung beginnt mit einem Startblock, der durch ein Raketen-Icon gekennzeichnet ist. Von hier aus entfaltet sich der Entscheidungsbaum deiner Anwendung.
2. Deine Anwendung endet mit einem oder mehreren Endblöcken. Diese verhindern, dass in der fertigen Anwendung ein "zum nächsten Schritt"-Button erscheint. Endblöcke werden im Blockmenü festgelegt.

### **Arten von Blöcken**

1. **Eingabe**: Diese Blöcke fragen Daten von den NutzerInnen ab. Die Eingaben können den Verlauf deiner Anwendung steuern und als Variablen wiederverwendet und gespeichert werden. Folgendes sind mögliche Eingabetypen:
   * Einfachauswahl: Ermöglicht Nutzer\:innen des interaktiven Formulars die Auswahl einer Antwort aus mehreren vordefinierten Antworten.
   * Mehrfachauswahl: Ermöglicht NutzerInnen des interaktiven Formulars die Auswahl einer Antwort aus mehreren vordefinierten Antworten.
   * Texteingabe: Eingabe einer Textzeile.
   * Textfeld: Eingabe mehrerer Textzeilen.
   * Nummerneingabe: Eingabe einer Zahl aus einem vordefinierten Zahlenbereich.
2. **Information**: Informationsblöcke stellen den NutzerInnen relevante Informationen zur Verfügung.
3. **Dokument**: Diese Blöcke führen eine Aktion aus, die ein Dokument basierend auf den Eingaben der Nutzer\:innen generiert.
4. **Report**: Reportblöcke versenden E-Mails mit einer Nachricht, einem Dokument-Anhang und/oder dem Entscheidungspfad an festgelegte Adressen.
5. **Logik**: Logikblöcke ermöglichen es, die Anwendung in Abhängigkeit von bestimmten Bedingungen zu verzweigen.

### **Block-Konfiguration und Verzweigung**

1. Jeder Block hat einen zugehörigen Editor, der geöffnet wird, indem du auf den Block klickst. Hier kannst du spezifische Einstellungen für den Block vornehmen und seine Verbindungen definieren.
2. Jeder Block kann sich verzweigen, das heißt, er kann mehr als eine Ausgangsverbindung haben. Dies ermöglicht die Erstellung eines Entscheidungsbaums in deiner Anwendung.
3. Die Bedingungen für die Aktivierung einer Verbindung werden im Block-Editor festgelegt. Du kannst neue Verbindungen erstellen, für jede Verbindung eine Bedingung und ein Ziel definieren.
4. Bedingungen bestehen aus einem Eingabewert, einem Vergleichsoperator und einem Vergleichswert. Der Eingabewert muss aus einer Eingabe innerhalb eines Blocks stammen, aber du kannst alle existierenden Eingaben abrufen.
5. Bedingungen können sehr komplex sein, indem du verschiedene Bedingungssätze miteinander verknüpfst. Du kannst beispielsweise zwei Bedingungen erstellen und zwischen ihnen die Option "Und" wählen, um eine komplexere Bedingung zu erstellen. Mit der Option "Oder" kannst du zwei Bedingungen unabhängig voneinander für eine Verbindung geltend machen.
